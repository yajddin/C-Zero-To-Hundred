# C Zero To Win

From getting started to the graveyard,It is kidding, please don't go to the graveyard!

> 从第一行 `Hello, C!` 开始，通过 200 道循序渐进的练习，从 C 语言入门一路走到多线程与现代 CMake 工程化。

## 项目简介

这是一个覆盖 C 语言基础与工程进阶的实践题库，共 200 道题，分为两条各 100 题的学习路线：

- [`Topics`](<Topics/001 Hello C/001 Hello C.md>)：从输入输出开始，覆盖分支、循环、数组、字符串、指针、结构体、文件和常见算法。
- [`Advanced Topics`](<Advanced Topics/001 指针读取整数/001 指针读取整数.md>)：从指针基础开始，深入内存管理、结构体设计、多线程和现代 CMake。

后面的题目会持续复用前面学过的知识，让你从“看得懂语法”走向“能独立完成工程”。

每道题都放在独立目录中，并提供：

- 题目描述
- 输入与输出格式
- 示例
- 边界条件与提示

项目默认只提供题目，不提供标准答案。请先独立思考、编写、测试和复盘。

## 适合谁

## 快速开始

### 1. 选择题目

初学者从 [Topics/001 Hello C](<Topics/001 Hello C/001 Hello C.md>) 开始；已经掌握 C 语法的读者可以从 [Advanced Topics/001 指针读取整数](<Advanced Topics/001 指针读取整数/001 指针读取整数.md>) 开始。

### 2. 编写程序

建议在对应题目目录中自行创建 `main.c`：

```text
Topics/
└── 001 Hello C/
    ├── 001 Hello C.md
    └── main.c
```

每道题尽量独立完成，不要直接复制上一题的完整代码。

### 3. 编译并运行

使用 GCC：

```powershell
gcc .\main.c -std=c11 -Wall -Wextra -O2 -o main.exe
.\main.exe
```

在 macOS 或 Linux 中：

```bash
cc main.c -std=c11 -Wall -Wextra -O2 -o main
./main
```

如果使用 Visual Studio、CLion、Code::Blocks 或其他 IDE，也可以直接创建 C 项目并运行。

进入进阶题库的 CMake 阶段后，请按照题目要求组织源码和构建目录。推荐始终使用源码外构建：

```powershell
cmake -S . -B build
cmake --build build
ctest --test-dir build --output-on-failure
```

### 4. 输入测试数据

题目示例中的 `输入：`、`输出：` 只是说明文字，不需要输入。程序运行后，只输入示例中的实际数据。

例如题目给出：

```text
输入：3 5
输出：8
```

运行程序时只输入：

```text
3 5
```

## 推荐做题方法

每道题都按下面的顺序完成：

1. **读题**：明确已知数据、目标结果和限制条件。
2. **手算**：不用代码，先手动算一遍示例。
3. **拆分**：把问题拆成输入、处理、输出三部分。
4. **编码**：先写出正确版本，再考虑简化和优化。
5. **编译**：开启警告，认真处理每一条警告。
6. **测试**：除了示例，还要主动测试边界情况。
7. **复盘**：记录错误原因和本题使用的知识点。

遇到困难时，可以按以下顺序处理：

1. 重新用自己的话描述题目。
2. 在纸上画出变量或数组的变化过程。
3. 把大问题缩小成只有 1～3 个数据的测试。
4. 查看题目提示，但先不要搜索完整答案。
5. 使用调试器或临时输出观察中间变量。

## 自测清单

提交或结束一道题前，检查：

- 输出格式是否完全一致，是否多了空格或文字
- 0、负数、最小规模和最大规模是否正确
- 重复元素、空字符串、空链表等特殊情况是否处理
- 整数是否可能溢出，是否需要 `long long`
- 浮点除法是否误写成整数除法
- 数组下标是否越界
- 字符串末尾是否保留 `\0`
- 动态内存是否申请成功并正确释放
- 文件是否成功打开，是否正确处理 `EOF`

## 文件题说明

基础题库第 091、092、093、099、100 题会读取或写入文件。运行程序前，请按题目要求把相应数据文件放到程序的工作目录中。

常见文件包括：

- `input.txt`
- `source.bin`
- `students.txt`
- `article.txt`

“工作目录”通常是你执行程序时所在的目录，不一定等于源代码所在目录。若程序提示文件打开失败，先检查文件名、扩展名和运行目录。

## 最后

C 语言的难点不在于记住多少语法，而在于能否把问题一步步变成程序。

不要等到“全部学会”再开始做题。写下第一版、看见错误、找到原因、重新运行——这正是学习编程最有效的过程。

现在，从 [001 Hello C](<Topics/001 Hello C/001 Hello C.md>) 开始；如果基础已经扎实，就直接挑战 [001 指针读取整数](<Advanced Topics/001 指针读取整数/001 指针读取整数.md>)。
